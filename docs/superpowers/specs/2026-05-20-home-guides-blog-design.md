# Home Guides → Blog-Driven Section — Design

**Date:** 2026-05-20
**Status:** Approved (design)
**Component:** Homepage "Guides" section (`patterns/home-guides.php`)

## Summary

The homepage Guides section currently renders hardcoded cards (static images, fixed
dates, fixed "5 min read", hardcoded chips) and its tabs only toggle a visual class
without changing content. The client has confirmed this content will be a **blog**.

This work makes the section data-driven from native WordPress posts: each tab is an
*audience*, each card is a real post showing its category, date, auto-calculated read
time, title, excerpt, and featured image. Tabs filter the cards instantly on the
client. Styling the blog's own archive/single pages is explicitly out of scope here.

## Decisions

| Decision | Choice |
|---|---|
| Content source | Native WordPress `post` posts (real blog) |
| Audience (tabs) | Custom `audience` taxonomy on `post` |
| Topic (chip) | Standard post Category (first assigned) |
| Tab switching | Instant client-side toggle (all sets rendered, non-active hidden) |
| Post selection | Newest 3 posts per audience; newest = feature card |
| Read time | Auto-calculated from content (~200 wpm, min 1) |
| Tabs source | Generated dynamically from `audience` terms that have posts |

## Data Model

### `audience` taxonomy
- Registered on the `post` post type, **hierarchical** (renders as checkboxes in the
  editor, matching the fixed-set selection intent).
- Seeded with three default terms on `init` if missing:
  - First-Time Visitor
  - Business Traveler
  - Weekend Getaway
- Slugs drive the tab/panel wiring (e.g. `first-time-visitor`, `business-traveler`,
  `weekend-getaway`).

### Chip (topic)
- Reads the post's **first standard Category** term name (Guide / Explore / Food / …).
- If the post has no category, the chip is omitted (no empty pill).

### Reading time
- Helper `hbd_reading_time( $post )`:
  `max( 1, ceil( str_word_count( wp_strip_all_tags( $content ) ) / 200 ) )`.
- Rendered as `"{n} min read"`.

### Card image
- Post **featured image** (`large` size).
- Fallback when none is set: existing `assets/images/guide-*.png` placeholder (or a
  neutral background block) so the layout never breaks.

## Architecture

### Data provider — `inc/guides-data.php`
New file (keeps `functions.php` lean), `require_once`'d from `functions.php` alongside
the other `inc/*` includes.

`hbd_get_guides_data()` returns, for each `audience` term that has **≥1 published post**,
a structure mirroring the existing `hbd_get_events_data()` convention:

```php
[
  'first-time-visitor' => [
    'term'    => WP_Term,                 // audience term (name + slug)
    'feature' => [ /* post view-model */ ], // slot 0 (newest)
    'cards'   => [ [ /* … */ ], [ /* … */ ] ], // slots 1–2
  ],
  // … one entry per non-empty audience, in term order
]
```

Each **post view-model** is pre-resolved so the template stays presentation-only:

```php
[
  'title'     => string,
  'permalink' => string (URL),
  'excerpt'   => string (trimmed; uses get_the_excerpt),
  'date'      => string (formatted, e.g. "14 March 2026"),
  'category'  => string ('' when none),
  'read_time' => string ("5 min read"),
  'image'     => string (featured image URL or fallback),
]
```

Query per audience: `WP_Query` with `post_type=post`, `posts_per_page=3`,
`orderby=date DESC`, `tax_query` on `audience` term, `post_status=publish`,
`ignore_sticky_posts=true`.

### Template — `patterns/home-guides.php`
Replace hardcoded markup with:

1. **Tabs** generated by looping the keys of `hbd_get_guides_data()`:
   ```html
   <div class="tabs" data-tabs role="tablist" aria-label="Guide audience">
     <button class="tab is-active" data-tab="{slug}" role="tab"
             aria-selected="true" aria-controls="guides-panel-{slug}">{Term name}</button>
     …
   </div>
   ```
   First tab is active.

2. **Panels** — one `home-guides__grid` per audience:
   ```html
   <div class="home-guides__grid" id="guides-panel-{slug}"
        data-panel="{slug}" role="tabpanel" {hidden unless first}>
     <div class="home-guides__feature"> {feature card — card--type-4} </div>
     <div class="home-guides__stack">
       {compact card — card--type-4 card--type-4-sm}   <!-- slot 1 -->
       {compact card — card--type-4 card--type-4-sm}   <!-- slot 2 -->
     </div>
   </div>
   ```
   - Card markup keeps the **existing classes** (`card--type-4`, `card--type-4-sm`,
     `tag-chip` variants, `card__meta`, etc.). Only the values become dynamic.
   - Feature card image + title link to the post permalink; compact cards link too.
   - Chip uses the post category; the two `tag-chip--small` meta pills use date and
     read time.

3. **"Read More" button** → `get_permalink( get_option( 'page_for_posts' ) )`, falling
   back to `home_url( '/' )` when no posts page is configured.

### Tab interaction — `src/js/modules/tabs.js`
Extend `initTabsRoot` so activating a tab also:
- Removes `is-active` from sibling tabs, sets `aria-selected` accordingly.
- Hides every `[data-panel]` in the section, then unhides the one whose `data-panel`
  matches the clicked tab's `data-tab`.
No AJAX — all panels are present in the DOM at load.

## Edge Cases

- **Audience with 0 posts:** no tab and no panel rendered for it.
- **Audience with 1–2 posts:** render the feature card plus whatever compact cards
  exist (right column shows 1 or 0 compact cards). Layout tolerates the gap.
- **No audiences have posts:** the entire section is not output.
- **Missing featured image:** placeholder fallback (above).
- **Missing category:** chip omitted.
- **More than 3 posts in an audience:** only newest 3 used.

## Out of Scope

- Styling/templating the blog archive (`index.php`) and single (`single.php`) pages
  beyond the existing bare scaffolding — handled later when a design exists.
- Pagination, search, or per-audience archive pages.
- Author, comments, related-posts UI.

## Affected / New Files

- **New:** `inc/guides-data.php` (taxonomy registration + term seeding + data provider
  + the `hbd_reading_time` helper — all three live here for now).
- **Edit:** `functions.php` (require the new include).
- **Edit:** `patterns/home-guides.php` (dynamic rendering).
- **Edit:** `src/js/modules/tabs.js` (panel toggling + ARIA).
- Possibly **edit:** `src/scss/sections/_home-guides.scss` only if hidden-panel /
  empty-slot states need adjustment (avoid scope creep otherwise).

## Acceptance Criteria

1. Creating a published post, assigning an audience and a category, and setting a
   featured image makes it appear in the matching tab on the homepage with correct
   title, excerpt, date, category chip, read time, and image.
2. The newest post in an audience is the large feature card; the next two are compact.
3. Clicking a tab instantly swaps the visible cards with no page reload and no network
   request; ARIA state updates correctly.
4. Audiences without posts produce no tab; an audience with fewer than 3 posts renders
   without layout breakage.
5. Read time matches `ceil(words / 200)` (min 1).
6. "Read More" navigates to the configured blog posts page.
